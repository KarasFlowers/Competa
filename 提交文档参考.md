# 二、功能说明

## 核心功能清单

**系统核心能力一览：**

- 多角色 Agent 协作：Collector、Survey、Interview、Fieldwork、Curator、Analyst、Writer、Screenshot、Filter、QA 十大 Agent 组成完整调研-分析-质检流水线

- 调研执行闭环：Survey/Interview Agent 产出问卷与访谈设计，Fieldwork Agent 模拟执行调研并将结果回流为可溯源证据，使"一手研究"真正进入分析链路

- 结构化知识抽取：预定义竞品知识 Schema（功能树 FeatureTree、定价模型 PricingModel、用户画像 Persona、SWOT 分析），通过 Guardrails 层强制校验确保输出一致性

- DAG 任务流转与反馈闭环：Agent 间通过 LangGraph StateGraph 编排，QA 质检不合格通过结构化 HandoffInstruction 打回至 Collector/Analyst/Writer 重做，最多 2 轮迭代

- 棘轮约束机制：QA 识别失败模式后自动转化为 ConstraintRule 持久化注入后续 Agent Prompt，确保同类错误不会重复发生

- 证据筛选与溯源：Curator 对来源去重、质量筛选、域名限额，Filter 过滤无引用 Claim 强制 Evidence 绑定；每条分析结论标注数据来源，支持一键跳转原始链接

- 可观测性：完整 Trace 系统记录每个 Agent 的决策过程与执行耗时，运行历史对比可量化改善幅度（evidence_coverage delta）

- 报告自动生成与导出：输出标准化 SWOT 分析报告与竞品对比矩阵，支持 Markdown / Word 导出

## 端到端使用流程

1. 用户在前端界面创建分析任务，输入目标产品、竞品列表、行业领域与分析关注点

2. Collector Agent 从公开渠道（URL / 文档搜索）多源采集竞品信息，遵守 robots.txt，采集结果持久化为 Source 记录

3. Survey Agent 根据任务上下文设计竞品分析问卷，Interview Agent 产出半结构化访谈提纲，两者承接维度一致

4. Fieldwork Agent 基于用户画像（Persona）与竞品上下文模拟执行问卷与访谈，产出标注 `[模拟]` 的可溯源 SURVEY/INTERVIEW 证据

5. Curator 证据筛选层对全部来源执行去重、低质量筛除、域名限额，生成可解释的筛选摘要（保留/剔除原因）

6. Analyst Agent 基于筛选后证据执行多维度对比分析，产出功能树、定价模型、用户画像与 SWOT 矩阵，持久化到 analyses 表

7. Writer Agent 将分析结果整合为结构化竞品报告，每条 Claim 强制绑定 Evidence ID

8. Screenshot Agent 补充竞品网页截图证据；Filter Agent 过滤无引用的 Claim

9. QA Agent 对报告进行事实校验与完整性审查，不合格则通过 HandoffInstruction 打回至 Collector/Analyst/Writer 重做（最多 2 轮），同时生成棘轮约束防止重复错误

10. 用户在前端查看完整报告，支持引用溯源跳转、Agent 决策回放、运行历史对比与 Markdown/Word 导出

## 大模型 / AI 能力使用说明

**模型调用**

- 适配层：统一 OpenAI 兼容接口（`llm/client.py`），通过环境变量 `LLM_BASE_URL` / `LLM_MODEL` 切换任意兼容供应商

- 默认模型：gpt-4o-mini（可根据实际部署环境配置为 DeepSeek / Doubao / 其他兼容模型）

- 调用方式：HTTP API + JSON Mode 强制结构化输出

- Token 预算机制：超出时自动裁剪低优先级内容

- 用途：信息采集与理解、问卷与访谈设计、竞品多维度对比分析、SWOT 生成、报告撰写、质检审查

**Agent 设计**

- 编排框架：LangGraph StateGraph + 条件边 + 确定性路由

- Agent 基类：`BaseAgent`（`agents/base.py`）统一封装 LLM 调用 + Schema 校验 + 重试逻辑（LLM 输出校验最多 3 次重试）

- 通信协议：Pydantic Schema 强类型约束 + Function Calling，所有 Agent 输入/输出通过 `schemas/` 模块定义

- 反馈机制：QA Agent 自动打回（最多 2 轮）+ 结构化 HandoffInstruction（目标 Agent、问题类型、失败字段、证据要求）+ 棘轮约束自动生成

- 调研闭环：Survey/Interview（设计）→ Fieldwork（执行模拟）→ 证据回流 Source 池 → Analyst 消费，让问卷与访谈产物真正影响分析结论

## 关键工程难点与解决方案

| 难点                  | 解决方案                                                                                                                 |
| ------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Agent 间上下文传递与状态管理   | 基于 LangGraph StateGraph 的 PipelineState 全局共享状态，10 个 Agent 节点间通过 Pydantic 强类型 Schema 传递结构化数据，Checkpoint 支持断点续传与运行历史对比 |
| 多 Agent 反馈闭环的无限循环风险 | 设置最大重试次数（2次）+ 确定性 qa_router 路由（非 LLM 决策）+ 渐进降级策略，超限后输出当前最优结果并标注置信度与失败原因                                              |
| 长文本信息的结构化抽取         | 设计分层 Schema（FeatureTree / PricingModel / Persona / SWOT）+ 分块处理策略，先粗粒度分类再细粒度提取，Guardrails 层自一致性校验（最多 3 次 LLM 重试）      |
| 证据质量管理与防止幻觉         | 三层防线：(1) Curator 证据筛选层（去重、质量评分、域名限额）；(2) Filter 过滤无 Evidence 绑定的 Claim；(3) QA 交叉验证事实性错误，棘轮约束防止同类错误重复                 |
| 调研产物悬空问题            | Survey/Interview 产出设计后，Fieldwork Agent 基于 Persona 模拟执行调研，明确标注 `[模拟]` 与 reliability 评分，将设计产物转为可溯源证据流入 Analyst 分析链路    |
| 前端重型可视化性能           | 路由级懒加载 + DAG 可视化（@xyflow/react）与对比矩阵（ComparisonMatrix）按需加载，降低首屏包体                                                    |

## 项目亮点 / 创新点

**亮点1：棘轮约束机制（Ratchet Constraint）**

QA 识别失败模式后，通过 `constraint_resolver.py` 自动将失败模式转化为 ConstraintRule 并持久化，后续重跑的 Agent Prompt 末尾自动注入这些约束，确保同类错误不会重复发生。这是现有主流 Agent 框架（LangGraph / CrewAI / OpenAI Agents SDK）均未提供的能力——它们支持打回重试，但不记忆"为什么失败"。

**亮点2：调研执行闭环**

区别于其他竞品分析工具仅依赖公开网页数据，Competa 通过 Survey → Interview → Fieldwork 三 Agent 链路，将问卷设计与访谈提纲"执行"为模拟调研证据（标注 `[模拟]` + reliability 评分），回流至证据池纳入分析——让"一手研究"设计真正对最终结论产生贡献，而非悬空的中间产物。

**亮点3：三层证据质量防线 + 全链路溯源**

Curator 筛选层（去重/质量/域名限额）+ Filter 过滤无引用 Claim + QA 交叉验证，构成三层防线。每条结论强制绑定 Evidence ID，支持前端一键跳转原始来源 URL。Agent 决策过程通过 DAG 回放可视化，全部执行 Trace 可追溯。
